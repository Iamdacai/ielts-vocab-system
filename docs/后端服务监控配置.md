## 📋 监控方案

### 方案一：Heartbeat 定期检查（推荐）

**配置位置**: `HEARTBEAT.md`  
**检查频率**: 每 30 分钟（heartbeat 触发时）  
**监控脚本**: `monitor-backend.sh`

**执行命令**:
```bash
/home/admin/.openclaw/workspace/git-repos/ielts-vocab-system/monitor-backend.sh check
```

**工作原理**:
1. Heartbeat 触发时执行监控脚本
2. 检查服务进程是否存在
3. 检查端口 3001 是否可访问
4. 如果服务停止，自动启动服务
5. 记录监控日志

**优点**:
- ✅ 与现有 heartbeat 机制集成
- ✅ 无需额外系统权限
- ✅ 可与其他检查任务一起执行
- ✅ 日志集中管理

### 方案二：Systemd 服务（生产环境）

**服务文件**: `ielts-vocab-backend.service`  
**配置位置**: `/etc/systemd/system/`

**安装步骤**:
```bash
# 复制服务文件
sudo cp ielts-vocab-backend.service /etc/systemd/system/

# 重载 systemd 配置
sudo systemctl daemon-reload

# 启用服务（开机自启）
sudo systemctl enable ielts-vocab-backend

# 启动服务
sudo systemctl start ielts-vocab-backend

# 查看状态
sudo systemctl status ielts-vocab-backend
```

**常用命令**:
```bash
# 启动服务
sudo systemctl start ielts-vocab-backend

# 停止服务
sudo systemctl stop ielts-vocab-backend

# 重启服务
sudo systemctl restart ielts-vocab-backend

# 查看日志
sudo journalctl -u ielts-vocab-backend -f

# 查看状态
sudo systemctl status ielts-vocab-backend
```

**优点**:
- ✅ 系统级守护进程
- ✅ 自动重启（`Restart=always`）
- ✅ 开机自启
- ✅ 资源限制和隔离
- ✅ 完整的日志管理

## 📊 监控指标

### 服务状态检查

| 检查项 | 方法 | 阈值 |
|--------|------|------|
| 进程状态 | `pgrep -f simple-server-https.js` | 存在 |
| 端口监听 | `curl -k https://localhost:3001/health` | HTTP 200 |
| 响应时间 | Health check 延迟 | < 3 秒 |
| 数据库连接 | Health check 返回 | total_words > 0 |

### 日志监控

**日志位置**:
- 服务日志：`backend/https-server.log`
- 监控日志：`backend/monitor.log`

**关键错误**:
- `Error: listen EADDRINUSE` - 端口被占用
- `Error: ENOENT: no such file or directory` - 证书文件缺失
- `SIGTERM` / `SIGKILL` - 服务被强制停止

## 🚨 告警配置

### 告警条件

当出现以下情况时发送告警：

1. **服务连续重启 3 次** - 可能存在配置问题
2. **服务无法启动** - 检查日志定位原因
3. **端口无响应** - 服务假死状态
4. **数据库连接失败** - 数据库文件损坏

### 告警方式

**飞书通知**（推荐）:

### 日常检查

```bash
# 查看服务状态
```

### 故障排查

**1. 服务无法启动**

```bash
# 检查端口占用
lsof -i :3001

# 检查证书文件
ls -la ssl/cert.pem ssl/key.pem

# 查看错误日志
tail -50 backend/https-server.log
```

**2. 服务频繁重启**

```bash
# 查看 systemd 日志
sudo journalctl -u ielts-vocab-backend -n 50

# 检查系统资源
free -h
df -h

# 检查数据库文件
ls -lh backend/scripts/ielts_vocab.db
```

**3. 证书过期**

```bash
# 检查证书有效期
openssl x509 -in ssl/cert.pem -noout -dates

# 重新生成证书（开发环境）
cd ssl
openssl req -x509 -nodes -days 365 -newkey rsa:2048 \
  -keyout server.key -out server.crt \
  -subj "/C=CN/ST=Beijing/L=Beijing/O=IELTS/CN=caiyuyang.cn"
```
