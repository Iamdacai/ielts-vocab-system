## 1. 项目背景

### 1.1 当前系统状态

**现有功能**：
- ✅ 首页九宫格可视化（艾宾浩斯 8 阶段）
- ✅ 复习页面（独立复习课模式）
- ✅ 发音练习功能（录音 + 评分）
- ✅ 多词库支持（剑桥雅思 + 真经词库）

**存在问题**：
- ❌ 九宫格在首页，复习页面独立，信息割裂
- ❌ 用户无法直观看到复习计划的整体分布
- ❌ 复习课入口不够清晰，需要理解两个页面的关系

### 1.2 优化目标

将**九宫格从首页迁移到复习页面**，实现：
1. 复习计划的可视化集中展示
2. 基于艾宾浩斯遗忘曲线的智能复习安排
3. 清晰的复习流程和用户引导

### 3.3 艾宾浩斯遗忘曲线逻辑

#### 3.3.1 复习阶段定义

| 阶段 ID | 阶段名称 | 复习间隔 | 颜色 | 说明 |
|--------|---------|---------|------|------|
| 0 | 新学 | 当天 | 🔴 红 | 首次学习的单词 |
| 1 | 第 1 天 | 1 天后 | 🟡 黄 | 第 1 次复习 |
| 2 | 第 2 天 | 2 天后 | 🟡 黄 | 第 2 次复习 |
| 3 | 第 4 天 | 4 天后 | 🟡 黄 | 第 3 次复习 |
| 4 | 第 7 天 | 7 天后 | 🟡 黄 | 第 4 次复习 |
| 5 | 第 15 天 | 15 天后 | 🟢 绿 | 第 5 次复习 |
| 6 | 第 21 天 | 21 天后 | 🟢 绿 | 第 6 次复习 |
| 7 | 已掌握 | 30 天后 | 🟢 绿 | 基本掌握 |

#### 3.3.2 阶段升降级规则

**答对（认识）**：
- 当前阶段 < 7：阶段 +1
- 当前阶段 = 7：保持阶段 7

**答错（不认识）**：
- 任何阶段：回退到阶段 0（重新学习）

**公式**：
```javascript
function calculateNextStage(currentStage, isCorrect) {
  if (isCorrect) {
    return Math.min(currentStage + 1, 7);
  } else {
    return 0;  // 答错直接回退到新学
  }
}
```

#### 3.3.3 复习日期计算

**公式**：
```javascript
function calculateNextReviewDate(lastReviewDate, stage) {
  const daysToAdd = REVIEW_STAGES[stage].days;
  const nextDate = new Date(lastReviewDate);
  nextDate.setDate(nextDate.getDate() + daysToAdd);
  return nextDate;
}
```

**示例**：
- 单词在 2026-03-15 学习（阶段 0）
- 下次复习：2026-03-16（阶段 1，1 天后）
- 如果答对：2026-03-18（阶段 2，2 天后）
- 如果答错：2026-03-16（阶段 0，重新学习）

### 3.4 每日复习课生成逻辑

#### 3.4.1 生成时机

**自动触发**：
- 每天凌晨 00:00 自动生成当日复习课
- 或者用户首次访问复习页面时动态计算

**推荐方案**：动态计算（避免定时任务复杂度）

#### 3.4.2 生成规则

```javascript
// 伪代码
function generateTodayReviewSession(userId) {
  const today = new Date();
  today.setHours(0, 0, 0, 0);
  
  // 查询所有 nextReviewDate <= today 的单词
  const dueWords = db.query(`
    SELECT w.*, p.stage, p.next_review_date
    FROM ielts_words w
    JOIN user_word_progress p ON w.id = p.word_id
    WHERE p.user_id = ?
      AND p.next_review_date <= ?
      AND p.status != 'mastered'
    ORDER BY p.next_review_date ASC
  `, [userId, today]);
  
  return {
    date: today,
    plannedWords: dueWords.length,
    words: dueWords,
    status: 'pending'
  };
}
```

#### 3.4.3 复习课状态

| 状态 | 说明 | 触发条件 |
|------|------|---------|
| pending | 未开始 | 复习课生成后，用户未开始 |
| in_progress | 进行中 | 用户点击"开始复习"后 |
| completed | 已完成 | 所有单词复习完成 |

## 4. 用户流程

### 4.1 主流程

```mermaid
graph TD
    A[用户打开小程序] --> B[首页]
    B --> C{点击去复习}
    C --> D[复习页面]
    
    D --> E[显示九宫格总览]
    E --> F[显示今日复习课信息]
    F --> G{有待复习单词？}
    
    G -->|是 | H[显示【开始复习】按钮]
    G -->|否 | I[显示"已完成今日复习"]
    
    H --> J[点击开始复习]
    J --> K[进入复习答题模式]
    
    K --> L[显示单词]
    L --> M[用户思考]
    M --> N{认识？}
    
    N -->|认识 | O[点击"认识"]
    N -->|不认识 | P[点击"不认识"]
    
    O --> Q[记录结果，阶段 +1]
    P --> R[显示答案，阶段归 0]
    
    Q --> S{还有单词？}
    R --> S
    
    S -->|是 | L
    S -->|否 | T[显示完成统计]
    
    T --> U[返回复习页面]
    U --> V[九宫格更新]
```

### 4.2 异常流程

| 场景 | 处理方式 |
|------|---------|
| 网络错误 | 提示"网络错误，请重试"，保留本地进度 |
| 中途退出 | 保存当前进度，下次继续 |
| 无复习任务 | 显示鼓励语，建议学习新词 |
| 复习课已完成 | 显示完成统计，允许重新复习 |

| updated_at | DATETIME | 更新时间 |

**索引**：
```sql
CREATE INDEX idx_user_review_date ON user_word_progress(user_id, next_review_date);
CREATE INDEX idx_user_stage ON user_word_progress(user_id, stage);
```

### 7.2 复习课表（✅ 需要实现）

**用于记录历史复习课数据**：

```sql
CREATE TABLE IF NOT EXISTS review_sessions (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  user_id INTEGER NOT NULL,
  session_date DATE NOT NULL,  -- 复习课日期
  planned_words INTEGER DEFAULT 0,  -- 计划单词数
  completed_words INTEGER DEFAULT 0,  -- 完成单词数
  status TEXT CHECK(status IN ('pending', 'in_progress', 'completed')),
  started_at DATETIME,  -- 开始时间
  completed_at DATETIME,  -- 完成时间
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  UNIQUE(user_id, session_date)
);

CREATE TABLE IF NOT EXISTS review_session_items (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  session_id INTEGER NOT NULL,
  word_id INTEGER NOT NULL,
  result TEXT CHECK(result IN ('known', 'unknown')),
  stage_before INTEGER,  -- 复习前阶段
  stage_after INTEGER,   -- 复习后阶段
  pronunciation_score INTEGER,  -- 发音评分（可选）
  answered_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (session_id) REFERENCES review_sessions(id),
  FOREIGN KEY (word_id) REFERENCES ielts_words(id)
);
```
