# 🔧 后端服务修复说明

**修复时间**: 2026-03-14 08:37  
**问题**: `ERR_CONNECTION_CLOSED` - HTTPS 连接失败

---

## ❌ 问题原因

**SSL 证书不存在**

后端服务尝试加载的 SSL 证书路径是软链接，指向 `/etc/letsencrypt/live/caiyuyang.cn/`，但该目录不存在或无权限访问。

```
backend/ssl/
├── cert.pem -> /etc/letsencrypt/live/caiyuyang.cn/ielts.crt  ❌ 不存在
└── key.pem -> /etc/letsencrypt/live/caiyuyang.cn/ielts.key   ❌ 不存在
```

**错误表现**:
```
POST https://caiyuyang.cn:3001/api/auth/login net::ERR_CONNECTION_CLOSED
```

---

## ✅ 修复方案

### 1. 创建自签名证书（开发环境）

```bash
cd /home/admin/.openclaw/workspace/git-repos/ielts-vocab-system/ssl

# 生成自签名证书
openssl req -x509 -nodes -days 365 -newkey rsa:2048 \
  -keyout server.key \
  -out server.crt \
  -subj "/C=CN/ST=Beijing/L=Beijing/O=IELTS/CN=caiyuyang.cn"
```

### 2. 更新软链接

```bash
cd /home/admin/.openclaw/workspace/git-repos/ielts-vocab-system/ssl

# 删除旧链接
rm -f cert.pem key.pem

# 创建新链接
ln -s server.crt cert.pem
ln -s server.key key.pem
```

### 3. 重启后端服务

```bash
# 停止旧服务
pkill -f "simple-server-https.js"

# 启动新服务
cd /home/admin/.openclaw/workspace/git-repos/ielts-vocab-system/backend
export PORT=3001
nohup node simple-server-https.js > https-server.log 2>&1 &

# 验证服务
curl -k https://localhost:3001/health
```

## ⚠️ 注意事项

### 自签名证书限制

**开发环境**: ✅ 可以使用自签名证书  
**生产环境**: ❌ 必须使用正式 SSL 证书

**原因**:
- 微信小程序要求 HTTPS
- 自签名证书在真机上会提示证书错误
- 生产环境需要使用 Let's Encrypt 或其他 CA 颁发的证书

### 生产环境部署

**获取正式证书**:
```bash
# 使用 Let's Encrypt (免费)
sudo certbot certonly --standalone -d caiyuyang.cn

# 证书路径
/etc/letsencrypt/live/caiyuyang.cn/fullchain.pem
/etc/letsencrypt/live/caiyuyang.cn/privkey.pem
```

**更新软链接**:
```bash
cd /home/admin/.openclaw/workspace/git-repos/ielts-vocab-system/ssl
rm -f cert.pem key.pem
ln -s /etc/letsencrypt/live/caiyuyang.cn/fullchain.pem cert.pem
ln -s /etc/letsencrypt/live/caiyuyang.cn/privkey.pem key.pem
```

## 🔍 常见问题排查

### 1. 服务无法启动

**检查端口占用**:
```bash
lsof -i :3001
# 或
netstat -tlnp | grep 3001
```

**解决方法**:
```bash
# 停止占用端口的进程
kill -9 <PID>
```

### 2. 证书加载失败

**检查文件权限**:
```bash
ls -la ssl/
# key.pem 应该是 600 权限
chmod 600 ssl/key.pem
```

### 3. 防火墙阻止连接

**开放端口**:
```bash
# Ubuntu/Debian
sudo ufw allow 3001/tcp

# CentOS/RHEL
sudo firewall-cmd --add-port=3001/tcp --permanent
sudo firewall-cmd --reload
```
