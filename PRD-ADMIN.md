## 1. 产品概述

### 1.1 产品定位
为雅思智能背单词系统提供 Web 端后台管理能力，支持管理员通过浏览器进行词库管理、用户管理、数据分析等核心运营工作。

### 1.2 目标用户
- 系统管理员（1-3 人）
- 运营人员（可选）

### 1.3 核心价值
- 🔐 安全可控：加密访问 + 账号权限管理
- 📊 数据可视化：直观的学习数据分析
- 🛠️ 高效运营：批量操作 + 自动化管理
- 📈 决策支持：用户行为分析指导产品优化

## 4. 技术架构

### 4.1 技术栈建议

| 层级 | 技术选型 | 说明 |
|------|----------|------|
| 前端 | Vue 3 + Element Plus | 快速开发，组件丰富 |
| 后端 | Node.js (Express/Koa) | 与现有后端技术栈一致 |
| 认证 | JWT + bcrypt | 成熟的认证方案 |
| 图表 | ECharts | 丰富的数据可视化 |
| 部署 | Nginx + HTTPS | 与现有服务统一部署 |

### 4.2 安全要求

- 🔒 **HTTPS 强制**：所有请求必须使用 HTTPS
- 🔐 **密码加密**：bcrypt 加密，salt rounds ≥ 10
- 🛡️ **XSS 防护**：前端输入过滤，后端输出转义
- 🚫 **CSRF 防护**：Token 验证
- 📝 **SQL 注入防护**：参数化查询
- 🔑 **会话安全**：JWT 过期时间、刷新机制
- 📊 **敏感数据脱敏**：导出时脱敏处理

### 4.3 接口设计

```
# 认证
POST   /admin/login          # 登录
POST   /admin/logout         # 登出
GET    /admin/me             # 获取当前管理员信息

# 词库管理
GET    /admin/wordbooks      # 词库列表
POST   /admin/wordbooks      # 创建词库
PUT    /admin/wordbooks/:id  # 更新词库
DELETE /admin/wordbooks/:id  # 删除词库
POST   /admin/wordbooks/:id/import  # 批量导入单词
GET    /admin/wordbooks/:id/words   # 词库单词列表

# 用户管理
GET    /admin/users          # 用户列表
GET    /admin/users/:id      # 用户详情
PUT    /admin/users/:id      # 更新用户
POST   /admin/users/:id/disable   # 禁用用户
POST   /admin/users/:id/enable    # 启用用户

# 统计分析
GET    /admin/stats/overview # 数据概览
GET    /admin/stats/users    # 用户分析
GET    /admin/stats/wordbooks # 词库分析
GET    /admin/stats/mistakes # 错题分析

# 系统配置
GET    /admin/config         # 获取配置
PUT    /admin/config         # 更新配置

# 操作日志
GET    /admin/logs           # 操作日志列表
```

## 8. 风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| 数据量大导致查询慢 | 中 | 添加索引、分页、缓存 |
